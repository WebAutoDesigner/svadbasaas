# Дизайн: модуль «Гости + RSVP»

Дата: 2026-06-01
Статус: утверждён пользователем, ждёт реализации (код пока не пишем)
Часть расширения функционала Svadba Plus (4 модуля: Гости+RSVP, Рассадка, База подрядчиков агентства, Календарь по всем свадьбам).

## Контекст и стратегия

Расширяем продукт после анализа рынка. Вывод: на рынке РФ нет специализированного
свадебного SaaS с кабинетом пары — это наш козырь для холодных продаж. Курс: углублять
кабинет пары + убирать ежедневную рутину агентства. В CRM (заявки, договоры, оплаты)
сознательно НЕ лезем — там доминируют amoCRM/Bitrix.

Список гостей — самый запрашиваемый модуль и фундамент для рассадки.

## Суть модуля

У каждой свадьбы есть список гостей. Ведут его и пара (в кабинете пары), и агентство
(координатор). Сами гости ничего не заполняют — статусы проставляются вручную
(ответы собирают по телефону/в мессенджерах). «RSVP» здесь = статус гостя, проставленный руками.

## Данные одного гостя

| Поле | Тип | Обязательность | Заметки |
|------|-----|----------------|---------|
| Имя | строка | обязательно | |
| Статус | enum: придёт / не придёт / под вопросом / не ответил | по умолчанию «не ответил» | |
| Сторона | enum: жених / невеста / общие | необязательно | для рассадки и баланса столов |
| Группа | строка (свободная метка) | необязательно | «родственники», «друзья», «коллеги» |

Привязка: гость принадлежит свадьбе (`weddingId`). Multi-tenant скоупинг — через `assertWedding`,
как у всех дочерних сущностей свадьбы.

## Экран «Гости»

- Таблица: имя · статус · сторона · группа
- **Счётчики сверху** (живой подсчёт): всего / придут / не придут / под вопросом / не ответили.
  Цифра «придут» — рабочая для кейтеринга.
- **Быстрый ввод**: набрал имя → Enter → строка добавилась. Цель — занести 100+ человек без боли.
- Фильтр и сортировка по стороне и группе.
- Смена статуса в один клик прямо в строке (inline).

## Права и роли

- Агентство (координатор) и пара — оба могут добавлять / править / удалять гостей.
- **Пометка об изменениях пары**: когда пара что-то меняет, у агентства на вкладке гостей
  загорается значок с числом непросмотренных правок + короткий список последних изменений
  («пара добавила Ивана», «пара удалила Петра»). Значок гаснет, когда координатор открыл список.
  - Реализация — поверх существующего `AuditLog` (мутации гостей уже логируются по паттерну проекта).
    Источник правок пары различаем по актору лога. «Последний просмотр» координатором — лёгкий
    маркер (отметка времени), сравниваем с временем правок пары.
  - **Без писем/пушей** — система уведомлений в скоуп не входит.

## Связь с рассадкой (следующий модуль)

Каждый гость — отдельная строка, поэтому рассадка возьмёт поимённо тех, у кого статус «придёт».
Группа/сторона помогут сажать рядом и балансировать столы.

## Явно НЕ делаем (зафиксировано с пользователем)

- Формы для гостей и публичной ссылки RSVP — нет (только ручной ввод).
- Меню / аллергий — нет.
- Пометки «ребёнок/взрослый» — нет.
- Писем-уведомлений — нет.
- Подсчёт безымянных «+N сопровождающих» — нет (каждый человек = отдельная строка).
