# Дизайн: модуль «Рассадка»

Дата: 2026-06-01
Статус: утверждён пользователем, ждёт реализации (код пока не пишем)
Зависит от модуля «Гости + RSVP» (см. 2026-06-01-guests-rsvp-design.md).

## Суть модуля

У каждой свадьбы — одна схема рассадки. Столы представлены карточками. Гостей
перетаскивают из бокового списка «не рассажены» на столы. Точность — **до стола**
(без нумерации стульев).

## Стол

| Поле | Тип | Заметки |
|------|-----|---------|
| Название | строка | «Стол 1» по умолчанию, можно переименовать («Молодожёны») |
| Число мест | число | вместимость |
| Порядок | число | сортировка карточек |

Привязан к свадьбе (`weddingId`). Операции: добавить / удалить / переименовать / задать места.

## Привязка гостя к столу

Гость (из модуля «Гости») получает необязательную привязку к столу (`tableId`, nullable).
Никакой отдельной сущности «место» — рассадка табличного уровня. Гость без `tableId` =
«не рассажен».

## Экран рассадки

```
┌─ Рассадка · Свадьба Ивановых ───────────── [⚠ пара внесла 2 правки] ┐
│ Столов: 8    Рассажено: 64 / 78                                      │
├─────────────────────┬───────────────────────────────────────────────┤
│ НЕ РАССАЖЕНЫ (14)    │  ┌ Стол 1 «Молодожёны» 6/8 ┐ ┌ Стол 2  8/10 ┐  │
│ ☑ только «придут»   │  │ • Иван Иванов           │ │ • …          │  │
│                     │  │ • Мария Иванова         │ │ • …          │  │
│ • Пётр Сидоров   ❓  │  │   ⟵ перетащи сюда       │ │              │  │
│ • Анна Кузнецова ✅  │  └─────────────────────────┘ └──────────────┘  │
│   (тащи на стол ⟶)  │  ┌ Стол 3  11/10 ⚠ ┐   [ + добавить стол ]     │
└─────────────────────┴───────────────────────────────────────────────┘
```

- **Карточка стола**: название, «занято / всего», список рассаженных гостей.
  Перебор мест (11/10) → подсветка ⚠, но **разрешён** (мягкое предупреждение, не блок).
- **Список «не рассажены»** слева: по умолчанию — **все** гости свадьбы.
  Галочка «только придут» фильтрует пул до статуса «придёт».
  У каждого гостя в пуле видно статус (✅ придёт / ❓ под вопросом / …), сторону и группу.
- **Действия**: перетащил гостя на стол — сел; перетащил обратно — вернулся в пул.
- Счётчики вверху (столов, рассажено / всего) — живые.

## Права и роли

- Агентство (координатор) и пара — оба перетаскивают гостей и правят столы.
- **Пометка об изменениях пары** — тот же механизм, что в модуле «Гости» (значок с числом
  непросмотренных правок + короткий список), на базе `AuditLog`. Без писем.
- Пара видит схему рассадки в своём кабинете.

## Явно НЕ делаем

- Нумерации стульев / конкретных мест — нет (только привязка к столу).
- Плана зала (сцена, танцпол, расстановка столов в пространстве) — нет.
- Нескольких вариантов/версий рассадки — нет (одна схема на свадьбу).
