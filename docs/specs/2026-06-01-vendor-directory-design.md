# Дизайн: модуль «База подрядчиков агентства»

Дата: 2026-06-01
Статус: утверждён пользователем, ждёт реализации (код пока не пишем)

## Суть модуля

Отдельный раздел на уровне агентства (не внутри свадьбы) — справочник проверенных
подрядчиков. Ведёт агентство. Пары к нему отношения не имеют. Цель — не вбивать
подрядчика заново на каждой свадьбе.

## Карточка подрядчика (AgencyVendor)

| Поле | Тип | Обязательность | Заметки |
|------|-----|----------------|---------|
| Имя | строка | обязательно | |
| Тип услуги | строка (свободная) | обязательно | фотограф / ведущий / площадка… (как существующий `Vendor.service`) |
| Контакт | строка | желательно | телефон/мессенджер/почта одной строкой |
| Ссылка | строка | необязательно | сайт / соцсети |
| Примерный прайс | строка | необязательно | ориентир «от 50т» (строка, не число — диапазоны/«договорная») |

Скоуп — по `agencyId` (через `withAgency`). Редактируют OWNER / ADMIN / COORDINATOR.

## Экран «Подрядчики» (раздел агентства)

- Список/таблица: имя · тип · контакт · ссылка · прайс
- Поиск по имени, фильтр по типу услуги
- Добавить / править / удалить

## Связь со свадьбой

При добавлении подрядчика на свадьбу (существующий per-wedding `Vendor`) появляется кнопка
**«выбрать из базы»** — выбор `AgencyVendor` префилит `name` / `service` / `contact` (и ссылку,
если решим хранить её на wedding-Vendor; иначе только три поля).

**Семантика — копия.** Данные копируются в wedding-`Vendor`. Дальнейшие правки подрядчика
на свадьбе не влияют на справочник и наоборот. Связи/истории нет.

Сумма (`amount`) и статус оплаты (`paymentStatus`) остаются wedding-specific, как сейчас.

## Явно НЕ делаем

- Истории «на каких свадьбах работал» — нет (нет связи directory↔weddings).
- Города/района — нет.
- Оценки/рейтинга/«проверенный» — нет.
- Отдельной заметки — нет.
- Доступа пары — нет (значка «изменения пары» здесь не существует).
