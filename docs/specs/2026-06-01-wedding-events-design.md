# Дизайн: модуль «События свадьбы»

Дата: 2026-06-01
Статус: утверждён пользователем, ждёт реализации (код пока не пишем)

Заменяет изначально планировавшийся «Календарь по всем свадьбам» — от кросс-свадебного
календаря отказались (слишком жидко: дат у задач чек-листа нет, привязывать нечего).
Вместо него — список дат-вех внутри конкретной свадьбы.

## Суть модуля

Внутри кабинета свадьбы (сторона агентства) — список мероприятий-вех. Русские свадьбы
часто растянуты (сборы, роспись/ЗАГС, венчание, банкет), каждое со своей датой и местом.

Отличие от существующего «тайминга дня» (`TimelineEvent`): тайминг — поминутный график
ОДНОГО дня (минуты от 00:00). «События» — ключевые вехи с ПОЛНЫМИ датами, возможно в
разные дни. Сущности держим раздельно.

## Карточка события (WeddingEvent)

| Поле | Тип | Обязательность | Заметки |
|------|-----|----------------|---------|
| Название | строка | обязательно | Сборы / Роспись / Венчание / Банкет… |
| Дата | дата | обязательно | UTC в БД, рендер по `wedding.timezone` (правило проекта) |
| Время | время | необязательно | роспись в 14:00; может отсутствовать |
| Описание | строка (free text) | необязательно | место + детали: «ЗАГС на Ленина, подъехать к 13:30» |
| Показать паре | bool | по умолчанию false | false = внутреннее (только агентство); true = пара видит |

Привязано к свадьбе (`weddingId`), скоуп через `assertWedding`.

## Экран «События» (сторона агентства)

```
┌─ События · Свадьба Ивановых ──────────── [ + добавить событие ] ┐
│ • Сборы невесты    18.10.2026 09:00   🔒 скрыто от пары          │
│     Квартира, визажист к 8:00                                    │
│ • Роспись          20.10.2026 14:00   👁 видна паре              │
│     ЗАГС на Ленина, подъехать к 13:30                            │
│ • Банкет           20.10.2026 17:00   👁 видна паре              │
│ ── прошедшие ──                                                  │
│ • Встреча с парой  02.06.2026   (приглушено)                     │
└──────────────────────────────────────────────────────────────────┘
```

- **Сортировка по дате**: ближайшие сверху; прошедшие — ниже и приглушены.
- Добавить / править / удалить — агентство (OWNER / ADMIN / COORDINATOR).
- Бейдж видимости (🔒 скрыто / 👁 видна паре) на каждом событии.

## Кабинет пары

Отдельный блок «Расписание свадьбы»: показывает **только** события с `показать паре = true`,
**read-only**, отсортированные по дате. Пара не создаёт и не правит → значка
«пара внесла изменения» здесь нет.

## Явно НЕ делаем

- Кросс-свадебного календаря (по всем свадьбам) — нет, отказались.
- Отдельного поля «место» — нет (пишется в описание).
- Напоминаний/уведомлений по событиям — нет.
- Правки событий парой — нет (пара только смотрит открытые ей).
